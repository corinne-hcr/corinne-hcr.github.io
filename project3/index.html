A.1 Shoot the Pictures

Photography Strategy: Images were captured by fixing the camera center of projection (COP) and rotating the camera to create panoramic sequences. Each set contains 2-3 images with 30-50% overlap between adjacent frames.

Image Resolution: All images were resized to a maximum width of 800 pixels before processing. This balances computation time (30 seconds vs 5-10 minutes at full resolution) with practical usability. However, at this resolution, selecting precise correspondence points becomes more challenging, as fine architectural details are smaller and less distinct.

A.2 Recover Homographies

Method Overview: The homography matrix H is computed using correspondence points manually selected between image pairs. The transformation is solved using least squares through Singular Value Decomposition (SVD).

Mathematical Formulation

Homography Transformation (Homogeneous Coordinates):

\[\begin{bmatrix} wx' \\ wy' \\ w \end{bmatrix} = \mathbf{H} \begin{bmatrix} x \\ y \\ 1 \end{bmatrix}\]

Where the homography matrix is:

\[\mathbf{H} = \begin{bmatrix} h_1 & h_2 & h_3 \\ h_4 & h_5 & h_6 \\ h_7 & h_8 & h_9 \end{bmatrix}\]

Dividing by w to get normalized coordinates:

\[x' = \frac{h_1 x + h_2 y + h_3}{h_7 x + h_8 y + h_9}\]
\[y' = \frac{h_4 x + h_5 y + h_6}{h_7 x + h_8 y + h_9}\]

Cross-multiplying to eliminate denominators:

\[x'(h_7 x + h_8 y + h_9) = h_1 x + h_2 y + h_3\]
\[y'(h_7 x + h_8 y + h_9) = h_4 x + h_5 y + h_6\]

Rearranging to move all terms to one side:

\[h_1 x + h_2 y + h_3 - h_7(xx') - h_8(yx') - h_9(x') = 0\]
\[h_4 x + h_5 y + h_6 - h_7(xy') - h_8(yy') - h_9(y') = 0\]

Multiplying by -1 to match standard form:

\[-h_1 x - h_2 y - h_3 + h_7(xx') + h_8(yx') + h_9(x') = 0\]
\[-h_4 x - h_5 y - h_6 + h_7(xy') + h_8(yy') + h_9(y') = 0\]

Linear System for each point correspondence:

For each point pair $(x, y) \rightarrow (x', y')$, we obtain two equations that can be written in matrix form:

\[\begin{bmatrix} -x & -y & -1 & 0 & 0 & 0 & xx' & yx' & x' \\ 0 & 0 & 0 & -x & -y & -1 & xy' & yy' & y' \end{bmatrix} \begin{bmatrix} h_1 \\ h_2 \\ h_3 \\ h_4 \\ h_5 \\ h_6 \\ h_7 \\ h_8 \\ h_9 \end{bmatrix} = \begin{bmatrix} 0 \\ 0 \end{bmatrix}\]

Full System with n correspondences:

\[\mathbf{A}\mathbf{h} = \mathbf{0}\]

Matrix A has shape (2n × 9). We need at least n ≥ 4 point pairs to solve for the 8 degrees of freedom (since h₉ is used for normalization).

Solution via Singular Value Decomposition (SVD):

\[\mathbf{A} = \mathbf{U} \mathbf{\Sigma} \mathbf{V}^T\]
\[\mathbf{h} = \text{last row of } \mathbf{V}^T \text{ (eigenvector for smallest singular value)}\]
\[\mathbf{H} = \text{reshape}(\mathbf{h}, 3 \times 3) / h_9 \text{ (normalize so } h_9 = 1\text{)}\]

A.3 Warp the Images

Warping Method: Images are warped using inverse warping to avoid holes in the output. Two interpolation methods are implemented and compared:
  • Nearest Neighbor: Fast but produces blocky results
  • Bilinear Interpolation: Smoother results with slight computational overhead

Observation: Bilinear interpolation produces noticeably smoother edges and reduces aliasing artifacts compared to nearest neighbor interpolation. The difference is particularly visible in diagonal lines and curved features.

Speed vs Quality Trade-off:

  • Nearest Neighbor: Fastest method (~10x faster than bilinear) but produces blocky, pixelated results with visible staircase artifacts on diagonal edges
  • Bilinear Interpolation: Slower but produces significantly smoother results. The quality improvement justifies the computational cost for final mosaics

For the final panoramas in Section A.4, bilinear interpolation was used exclusively to ensure smooth blending.

A.4 Blend the Images into a Mosaic

Blending Strategy: A one-shot warping approach is used where all images are warped to a common projection plane. Alpha masks with linear feathering are created on the original images before warping, then weighted averaging is applied to blend overlapping regions smoothly.

Key Steps:

  1. Select middle image as reference frame (unwarped coordinate system)
  2. Compute transform matrices for all other images relative to reference
  3. Create alpha masks (center=1, edges=0) on original images before warping
  4. Warp all images (including reference with identity transform) with their alpha channels using bilinear interpolation
  5. Blend using weighted average: Color = Σ(Image × Alpha) / Σ(Alpha)

Alpha Blending Formula:

\[I_{\text{blend}}(x,y) = \frac{\sum_i \alpha_i(x,y) \cdot I_i(x,y)}{\sum_i \alpha_i(x,y)}\]

Part B: Feature Matching for Autostitching

B.1 Harris Corner Detection and ANMS

Corner Detection Pipeline:
  1. Harris Corner Detection: Identify interest points using the Harris corner detector
  2. Adaptive Non-Maximal Suppression (ANMS): Select well-distributed corners by computing suppression radius for each corner. Each corner's radius is the distance to the nearest corner that is significantly stronger (using robustness constant c=0.9). Select the top 500-800 corners with largest suppression radii.

Before ANMS (thousands of corners) vs After ANMS (800 well-distributed corners)

ANMS Benefits: The suppression algorithm ensures corners are spatially distributed across the image rather than clustered in high-texture regions. This improves matching quality by providing coverage of the entire scene.

B.2 Feature Descriptor Extraction

Descriptor Extraction Process:
  1. Extract 40×40 pixel patches around each corner from a blurred image (σ=2.0)
  2. Downsample to 8×8 descriptors by averaging 5×5 pixel blocks
  3. Normalize each descriptor to zero mean and unit variance

This creates rotation-invariant, bias/gain normalized feature descriptors robust to illumination changes.

B.3 Feature Matching

Lowe's Ratio Test: For each feature in image 1, find the two nearest neighbors in image 2 based on descriptor distance. Accept a match only if the ratio of distances (1-NN / 2-NN) is below threshold (0.5-0.7). This ensures distinctiveness and reduces false matches.

B.4 RANSAC for Robust Homography Estimation

RANSAC Algorithm:
  1. Randomly sample 4 point correspondences
  2. Compute homography H from these 4 points
  3. Count inliers (points with reprojection error < threshold)
  4. Repeat for many iterations (2000-5000)
  5. Select H with most inliers and recompute using all inliers

Parameters used: threshold = 2.0 pixels, iterations = 5000

Results Analysis: The automatic stitching pipeline produces results comparable to manual correspondence selection. RANSAC effectively filters most outliers from the feature matches. However, some inliers may still come from challenging regions like ground textures, vegetation, or sky, which lack distinct geometric features and are inherently difficult to align accurately. In such cases, the automated approach may not always outperform careful manual point selection. The automated method is faster and more reproducible, making it practical for most scenarios, while manual selection can achieve better precision when dealing with specific difficult features.
